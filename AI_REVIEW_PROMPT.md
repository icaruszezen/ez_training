## 一、项目概况（上下文）

本项目 **EZ Traing** 是一个基于 **Python 3 + PyQt5 + PyQt-Fluent-Widgets** 的桌面应用，用于目标检测的标注与训练一体化。

**技术栈：**
- GUI：PyQt5、PyQt-Fluent-Widgets（Fluent Design 风格）
- 目标检测：Ultralytics YOLO
- 图像处理：OpenCV、Pillow、Albumentations
- 数据格式：VOC (XML) / YOLO (TXT)
- 其他：lxml、Matplotlib、PyYAML、requests

**核心目录结构：**
```
src/ez_traing/
├── main.py              # 入口
├── dep_installer.py     # 运行时依赖安装
├── updater.py           # GitHub Releases 自动更新
├── ui/                  # 主窗口、绘制、后台线程
├── pages/               # 各功能页面
├── prelabeling/         # 预标注（视觉 API + YOLO）
├── labeling/            # labelImg 风格标注
├── template_matching/   # 模板匹配
├── data_prep/           # 数据准备流水线
├── evaluation/          # 验证与报告
├── annotation_scripts/  # 内置脚本示例
└── common/constants.py  # 常量、配置目录、设备检测
tests/                   # 单元测试
```

## 二、全面分析提示词

请将以下内容发送给 AI：

---

### 提示词开始 ↓

你是一名资深 Python 桌面应用开发工程师，精通 PyQt5、计算机视觉和软件架构设计。请对本项目进行全面审查，按以下维度逐一分析，并给出具体的修复方案，进行代码修改。

#### 1. 代码质量与规范

- 检查所有 Python 文件是否遵循 PEP 8 规范
- 识别不一致的命名风格（变量、函数、类）
- 查找重复代码，提出提取公共方法/工具类的建议
- 检查 import 语句：是否有未使用的导入、循环导入、导入顺序不规范
- 检查类型注解的完整性和正确性

#### 2. Bug 与潜在错误

- 查找所有未处理的异常（bare except、过宽的 except）
- 检查空值/None 未检查就使用的情况
- 检查文件操作是否使用了 with 语句或正确关闭资源
- 检查多线程/QThread 中的线程安全问题（信号槽、共享变量）
- 检查路径处理是否跨平台兼容（os.path vs pathlib）
- 检查数组越界、字典 KeyError 等运行时风险
- 检查 PyQt5 信号槽连接是否正确、是否有内存泄漏

#### 3. GUI 与用户体验

- 检查所有 UI 操作是否在主线程执行（不在工作线程直接操作 UI）
- 检查耗时操作是否放在后台线程（避免界面卡顿）
- 检查进度条/状态反馈是否完整
- 检查错误提示是否对用户友好
- 检查窗口关闭、取消操作时的资源清理

#### 4. 架构与设计

- 分析模块间耦合度，识别过度耦合
- 检查是否有 God Class（过大的类承担了太多职责）
- 检查页面/组件之间的数据传递是否合理
- 分析 LazyPageHost 懒加载机制是否可能导致问题
- 检查配置管理（settings.json、datasets.json）的读写安全性

#### 5. 性能问题

- 检查大数据集/大图片的加载是否有分页或懒加载
- 检查缩略图生成是否有缓存机制
- 检查是否有不必要的重复计算
- 检查内存占用（大图片是否及时释放）
- 检查文件 I/O 是否高效

#### 6. 安全性

- 检查外部输入（文件路径、API 参数）是否做了验证和清理
- 检查 API key 等敏感信息是否安全存储
- 检查脚本执行功能（annotation_scripts）的安全风险
- 检查自动更新（updater.py）的安全性

#### 7. 测试覆盖

- 分析当前测试覆盖率，指出哪些关键模块缺少测试
- 为未覆盖的关键路径建议测试用例
- 检查现有测试是否有效（是否真正验证了行为）

#### 8. 依赖管理

- 检查 requirements.txt 中的依赖版本是否合理
- 检查是否有已知漏洞的依赖版本
- 分析运行时依赖安装机制（dep_installer.py）的鲁棒性
- 检查 PyInstaller 打包配置（ez_traing.spec）是否完整

---

**输出格式要求：**

对于发现的每个问题，请按以下格式输出：

```
### [严重程度: 高/中/低] 问题标题

**文件：** `路径/文件名.py`
**行号：** 第 XX 行
**问题描述：** 简要说明问题
**风险：** 可能导致的后果
**修复方案：**
（给出具体的代码修改，使用 diff 格式或完整替换代码）
```

请按严重程度从高到低排序输出。先列出所有高严重度问题，再列出中等，最后列出低严重度。

## 四、使用建议

1. **分步执行**：项目文件较多，建议先用"全面分析"做整体扫描，再用针对性提示词深入排查。
2. **提供上下文**：将本文件第一节"项目概况"始终包含在提示词中，帮助 AI 快速理解项目。
3. **逐模块分析**：如果 AI 上下文窗口有限，可以按模块分批分析：
   - 第 1 批：`main.py`、`ui/`、`pages/`
   - 第 2 批：`prelabeling/`、`labeling/`
   - 第 3 批：`template_matching/`、`data_prep/`
   - 第 4 批：`evaluation/`、`annotation_scripts/`、`common/`
4. **验证修复**：AI 给出修复代码后，运行 `pytest` 确认没有引入新问题。
5. **持续使用**：每次大版本迭代后，重新运行一次全面分析。
