## 项目背景

**EZ Traing (FluentLabel)** — 基于 Python 3 + PyQt5 + PyQt-Fluent-Widgets 的目标检测标注与训练一体化桌面工具。

| 项目 | 说明 |
|------|------|
| GUI | PyQt5、PyQt-Fluent-Widgets (Fluent Design) |
| 目标检测 | Ultralytics YOLO、PyTorch |
| 图像处理 | OpenCV、Pillow、Albumentations |
| 数据格式 | VOC (XML) / YOLO (TXT) / CreateML |
| 打包 | PyInstaller（torch/ultralytics 放 deps/） |

**核心目录：**
```
src/ez_traing/
├── main.py              # 入口，deps 加载
├── ui/                  # 主窗口、绘图画布、后台线程
├── pages/               # 各功能页面（数据集、标注、训练等）
├── labeling/            # labelImg 风格标注核心
├── prelabeling/         # 预标注（YOLO / Vision API）
├── template_matching/   # 模板匹配
├── data_prep/           # 数据准备流水线
├── evaluation/          # 模型验证与报告
├── annotation_scripts/  # 内置脚本标注
└── common/constants.py  # 常量、配置、设备检测
tests/                   # 单元测试
```

### 第一阶段：全局扫描

逐一阅读 `src/ez_traing/` 下的所有 Python 文件，建立整体理解。扫描时记录以下类别的问题：

**P0 — 崩溃级（必须修复）：**
- 未捕获异常导致程序崩溃
- None / 空值未检查直接使用
- 数组越界、字典 KeyError
- 信号槽连接错误导致功能失效
- 文件/资源句柄未关闭导致泄漏
- 多线程中直接操作 UI 导致段错误

**P1 — 功能缺陷（应当修复）：**
- 线程安全问题（QThread 中共享变量无保护）
- 取消/关闭操作未正确清理资源
- 路径处理不兼容（硬编码分隔符、未处理中文/空格路径）
- 错误处理过宽（bare except / except Exception 吞掉所有异常）
- 进度反馈缺失或不准确

**P2 — 代码质量（建议修复）：**
- 重复代码可提取为公共方法
- 未使用的导入
- 命名不规范
- 类型注解缺失导致的可读性问题
- 过大的类/函数需要拆分

**P3 — 性能优化（酌情修复）：**
- 大图片/大数据集未做懒加载或分页
- 缩略图未缓存
- 不必要的重复计算
- 主线程执行耗时操作导致界面卡顿

### 第二阶段：逐一修复

扫描完成后，按 P0 → P1 → P2 → P3 的优先级顺序逐一修复。

**对每个修复，执行以下步骤：**

1. **说明问题**（一句话描述问题和风险）
2. **定位代码**（文件路径 + 行号范围）
3. **直接修改代码**（不要只给建议，直接编辑文件）
4. **解释修改**（简要说明改了什么、为什么这样改）

**修复时遵守的原则：**

- **最小改动原则**：只修改必要的部分，不做大规模重构，除非问题本身要求
- **向后兼容**：不改变公共 API 的签名和行为，除非是修 Bug
- **保持风格一致**：遵循项目现有代码风格（命名、缩进、import 顺序）
- **不引入新依赖**：修复应使用项目已有的库
- **异常处理**：用具体的异常类型替代 bare except，添加有意义的日志
- **线程安全**：涉及 QThread 的修复，确保 UI 操作通过信号槽
- **路径处理**：统一使用 `pathlib.Path` 或 `os.path`，不混用
- **资源管理**：文件操作使用 `with` 语句
